2. Conceptos Principales - Base de Datos

  • Firmware: El Firmware es un tipo de software especializado que está incrustado directamente en el hardware de un dispositivo. A diferencia del software tradicional que se carga en la memoria RAM, el firmware reside en una memoria no volátil, como ROM, EPROM o Flash. Su función principal es controlar las operaciones de bajo nivel del hardware, actuando como un puente entre el sistema operativo y los componentes físicos.
  • Middleware: El Middleware es una capa de software que se sitúa entre diferentes aplicaciones, sistemas operativos o bases de datos, facilitando su comunicación e interacción. Su propósito es simplificar el desarrollo de aplicaciones distribuidas, permitiendo que componentes de software heterogéneos trabajen juntos sin necesidad de que los desarrolladores gestionen directamente las complejidades de la red o los protocolos de comunicación.
  • Cuantificación de bits y bytes: La cuantificación de bits y bytes es fundamental para entender cómo se representan y miden los datos digitales. La unidad más básica de información es el bit, que es un dígito binario (0 o 1). Un bit por sí solo no puede representar mucha información. Aquí es donde entra el byte. Un byte se compone de 8 bits. Al agrupar los bits de esta manera, se pueden representar un rango mucho mayor de valores. Por ejemplo, un byte puede representar 256 valores diferentes (desde 0 hasta 255), lo que es suficiente para codificar un solo carácter, como una letra o un número.
  • Arquitectura de un Sistema Operativo: La arquitectura de un Sistema Operativo (SO) se refiere a la estructura interna y la organización de sus componentes, los cuales trabajan en conjunto para gestionar los recursos del sistema y proporcionar una interfaz para el usuario y las aplicaciones. Aunque existen diferentes tipos de arquitecturas, la mayoría de los SO modernos siguen un modelo por capas o modular.
  • Bases de Datos en MySQL: MySQL es uno de los Sistemas de Gestión de Bases de Datos Relacionales (SGBDR) de código abierto más populares y ampliamente utilizados en el mundo. Su popularidad se debe a su fiabilidad, rendimiento, facilidad de uso y la gran comunidad de desarrolladores que lo soporta. En un SGBDR como MySQL, los datos se organizan en tablas, que son colecciones de filas y columnas. Cada tabla representa una entidad (por ejemplo, "Clientes" o "Productos"), y cada fila en la tabla representa una instancia de esa entidad. Las columnas definen los atributos de la entidad (por ejemplo, "Nombre", "Dirección", "Precio").

3. Componentes de Comunicación

  • Sistema de Bases numéricas: Los sistemas de bases numéricas son diferentes maneras de representar cantidades. Los más comunes en informática son:

    Binario (Base 2): Utiliza solo dos dígitos: 0 y 1. Es el lenguaje fundamental de las computadoras, ya que se corresponde directamente con los estados de encendido/apagado (o verdadero/falso) de los componentes electrónicos.

    Decimal (Base 10): Es el sistema que usamos en la vida diaria, con diez dígitos del 0 al 9.

    Hexadecimal (Base 16): Utiliza 16 símbolos: del 0 al 9 y las letras A a F. Es muy usado en programación y redes para representar valores binarios grandes de forma más compacta y legible. Por ejemplo, un dígito hexadecimal puede representar exactamente 4 bits.

  • Introducción a Redes: Una red es simplemente un conjunto de dispositivos interconectados que pueden compartir recursos e información. Estos dispositivos pueden ser computadoras, servidores, impresoras, teléfonos inteligentes, etc. La conexión puede ser física (con cables) o inalámbrica. El objetivo principal de una red es facilitar la comunicación y el intercambio de datos entre sus componentes
  • Implementación de Redes: La implementación de redes abarca todo el proceso de hacer que una red funcione. Esto incluye varias fases: Diseño: Planificar la topología de la red, seleccionar los equipos adecuados (routers, switches, cables, etc.) y decidir cómo se conectarán los dispositivos. Instalación: El montaje físico de los equipos y el tendido de los cables. Configuración: Establecer los parámetros de software en los dispositivos de red, como direcciones IP, protocolos de enrutamiento y medidas de seguridad, para asegurar que los datos fluyan correctamente.
  • Manejo de octetos: Un octeto es una unidad de información digital equivalente a 8 bits. Es sinónimo de "byte", aunque el término "octeto" se prefiere en contextos de redes y telecomunicaciones para evitar cualquier ambigüedad, ya que históricamente en algunos sistemas un "byte" no siempre era de 8 bits.

4. Resolución de Problemas y Pseudocódigo

  • Planteamiento y Resolución de Problemas: Identificar, analizar y solucionar problemas paso a paso...
  • Algoritmos y Diagramas: Una vez que se tiene una idea de la solución, se formaliza a través de algoritmos y diagramas: Un algoritmo es una secuencia finita y bien definida de pasos lógicos y ordenados para resolver un problema o realizar una tarea. Piense en ello como una receta detallada. Los algoritmos se pueden representar de varias maneras, incluyendo:

    Pseudocódigo: Una descripción de alto nivel de un algoritmo que se asemeja a un lenguaje de programación, pero está diseñado para ser legible por humanos y no tiene una sintaxis estricta. Es ideal para planificar la lógica antes de escribir código real.

    Diagramas de flujo: Representaciones gráficas de los pasos de un algoritmo, utilizando símbolos estandarizados para diferentes tipos de acciones (inicio/fin, proceso, decisión, entrada/salida).

  • Tipos de Operadores: Los operadores son símbolos especiales que se utilizan para realizar operaciones sobre uno o más valores (llamados operandos). Son esenciales para construir expresiones lógicas y matemáticas en los algoritmos y programas:

    Aritméticos: Realizan operaciones matemáticas básicas como suma (+), resta (-), multiplicación (*), división (/) y módulo (%).

    Relacionales: Comparan dos valores y devuelven un resultado booleano (verdadero o falso), como igual a (==), diferente de (!=), mayor que (>), menor que (<), mayor o igual que (>=), menor o igual que (<=).

    Lógicos: Combinan o modifican expresiones booleanas, como AND (&& o AND), OR (|| o OR) y NOT (! o NOT). De asignación: Se utilizan para asignar un valor a una variable, siendo el más común el signo igual (=). También existen combinados como +=, -=, etc. Otros: Dependiendo del lenguaje, puede haber operadores de incremento/decremento (++, --), de concatenación de cadenas, bit a bit, entre otros.

  • Constantes, Variables, Tipos de Instrucciones: Estos son los bloques de construcción fundamentales en cualquier lenguaje de programación:

    Constantes: Son valores fijos que no cambian durante la ejecución de un programa. Por ejemplo, el valor de PI (3.14159) podría ser una constante.

    Variables: Son espacios de almacenamiento en la memoria de la computadora que pueden contener datos. Su valor puede cambiar durante la ejecución del programa. Las variables tienen un nombre, un tipo de dato (por ejemplo, número entero, texto, booleano) y un valor.

    Tipos de Instrucciones: Se refieren a los diferentes comandos o sentencias que un programa puede ejecutar. Incluyen:

    Instrucciones de entrada/salida: Para que el programa interactúe con el usuario (leer datos, mostrar resultados).

    Instrucciones de asignación: Para guardar valores en variables.

    Instrucciones de control de flujo: Como condicionales (ej. SI... ENTONCES... SINO) y bucles (ej. PARA, MIENTRAS), que dictan el orden en que se ejecutan las otras instrucciones
